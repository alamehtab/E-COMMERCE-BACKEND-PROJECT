STEP 1 ---> USER ADDS PRODUCT TO CART  POST /api/cart/add
where price is saved with product.price.

STEP 2 ---> USER CLICKS CHECKOUT   POST api/payment/createPayment
where totalPrice is calculated and razorpay order is created and razorpay return an order_id. Backend saves order_id as razorrpayOrderId. Note: Payment is not made yet. Backend send response with order_id, amount, currency and razorpay_api_key. Key and id is used by the frontend to open razorpay popup and checkout ui.
razorpay usually returns an object after creating payment order as shown below:
{
  id: "order_Mx7abc123xyz",
  entity: "order",
  amount: 50000,
  amount_paid: 0,
  amount_due: 50000,
  currency: "INR",
  receipt: "receipt_1712400000000",
  status: "created",
  attempts: 0,
  created_at: 1712400000
}

STEP 3 ---> FRONTEND OPEN RAZORPAY CHECKOUT
where enters payment details for upi, card, etc.

STEP 4 ---> USER PAYS MONEY
where backend is not invovled and razorpay confirms payment with webhooks and 

STEP 5 ---> RAZORPAY RETURNS PAYMENT DETAILS
which contains razorpay_order_id, razorpay_payment_id and razorpay_signature.

STEP 6 ---> FRONTEND SEND PAYMENT DETAILS AS A PROOF OF REAL PAYMENT
where frontend calls POST /verifypayment and send razorpay_order_id, razorpay_payment_id and razorpay_signature.

STEP 7 ---> BACKEND VERIFIES PAYMENT 
STEP 8 ---> BACKEND UPDATES PAYMENT RECORDS 
STEP 9 ---> CONVERTS CART INTO ORDER AND CREATES ORDER
STEP 10 ---> REDUCE STOCK AND CLEAR CART

OVERALL STRUCTURED WORK FLOW
       ↓↓↓
Add Product to Cart
        ↓
Price locked in cart
        ↓
User clicks Checkout
        ↓
Frontend → createPayment API
        ↓
Backend creates Razorpay Order
        ↓
Payment record saved (pending)
        ↓
Frontend opens Razorpay Checkout
        ↓
User completes payment
        ↓
Razorpay returns payment data
        ↓
Frontend → verifyPayment API
        ↓
Backend verifies signature (crypto)
        ↓
Payment marked success
        ↓
Order created
        ↓
Stock reduced
        ↓
Cart cleared
